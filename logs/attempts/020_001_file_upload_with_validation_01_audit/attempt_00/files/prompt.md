Write me an Elixir application composed of a few modules that implements a file upload endpoint with validation, using only `Plug` (no Phoenix). The main entry point is a `Plug.Router` module called `FileUpload.Router` that exposes `POST /api/uploads`.

Here's what I need:

**`FileUpload.Router`** — a `Plug.Router` that:

- Parses multipart uploads via `Plug.Parsers` with a `:length` limit of 5MB (5_242_880 bytes). If a request exceeds this, return HTTP 413 with JSON body `{"error": "File too large", "max_bytes": 5242880}`.
- Accepts a single file upload under the form field name `"file"`.
- Delegates validation to `FileUpload.Validator` and storage to `FileUpload.Store`.
- On success, returns HTTP 201 with JSON metadata: `{"id", "original_name", "size", "content_type", "uploaded_at", "download_url"}`.
- On validation failure, returns HTTP 422 with JSON body `{"error": "<descriptive message>"}`.
- If the `"file"` field is missing, return HTTP 422 with `{"error": "No file provided"}`.

The router must accept these options at init time via `plug FileUpload.Router, opts`:
- `:store` — the PID or name of the `FileUpload.Store` GenServer.
- `:upload_dir` — the directory path where files are saved to disk.
- `:base_url` — the URL prefix for generating download URLs (e.g. `"http://localhost:4000"`).

**`FileUpload.Validator`** — a module with a single public function:

- `validate(upload)` where `upload` is a `%Plug.Upload{}` struct. It returns `:ok` or `{:error, reason_string}`.
- It must enforce:
  1. **File type**: only `.csv` and `.json` extensions are allowed (check the `filename` field, case-insensitive). If not, return `{:error, "File type not allowed. Only .csv and .json files are accepted"}`.
  2. **Content validity for CSV**: read the file at `upload.path`, parse it enough to check it has at least one row that looks like a header (i.e., at least two lines or at least one line with comma-separated values). If empty or a single value on a single line, return `{:error, "Invalid CSV: file must contain a header row with multiple columns"}`.
  3. **Content validity for JSON**: read the file and attempt `Jason.decode`. If it fails, return `{:error, "Invalid JSON: " <> description}` where description comes from the decode error.

**`FileUpload.Store`** — a GenServer that:

- `start_link(opts)` accepts a `:name` option for process registration.
- `save(server, metadata_map)` stores file metadata keyed by `id`. The `id` should be a UUID v4 string generated by the store itself. It should add an `uploaded_at` field as an ISO 8601 UTC timestamp string. Returns `{:ok, full_metadata_with_id_and_timestamp}`.
- `get(server, id)` retrieves stored metadata. Returns `{:ok, metadata}` or `{:error, :not_found}`.
- `list(server)` returns all stored metadata as a list.

When saving a file to disk, the router should use the generated UUID as the filename (preserving the original extension) so files don't collide.

Use `Jason` for JSON encoding/decoding. Use only standard OTP plus `Plug` and `Jason` as dependencies — no Phoenix, no Ecto, no database. Keep everything in a single file if possible, but clearly separated into the three modules. Include `@moduledoc` for each module.
